Ты делаешь refactoring.

Цель:
Улучшить структуру кода без изменения поведения.

Правила:
- поведение системы должно остаться прежним;
- не добавляй новую feature;
- не меняй внешние API без причины;
- не смешивай refactoring и bug fixing;
- не переименовывай всё подряд;
- делай маленькие понятные изменения.

Перед изменениями:

## Scope
Работай только с указанными файлами.

## Refactoring goal
Что именно улучшаем:
- читаемость;
- разбиение функций;
- устранение дублирования;
- улучшение зависимостей;
- упрощение логики.

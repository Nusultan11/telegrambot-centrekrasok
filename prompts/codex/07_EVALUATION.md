Ты работаешь с evaluation.

Цель:
Проверять качество RAG и LLM-ответов не вручную, а через набор тестовых вопросов.

Что нужно оценивать:
1. Retrieval quality
Нашёл ли retriever правильный контекст.

2. Answer faithfulness
Ответ основан на контексте или модель выдумала.

3. Answer relevance
Ответил ли бот на вопрос пользователя.

4. No-answer behavior
Правильно ли бот отказывается отвечать, если данных нет.

Минимальный evaluation dataset:

question | expected_context_keywords | expected_behavior

Примеры:
- Чем занимается компания?
- Какие услуги предоставляет?
- Где находится офис?
- Какие технологии используются?
- Какие есть вакансии?
- Сколько сотрудников работает в компании?
- Кто директор компании?
- Какая зарплата у сотрудников?

Правила:
- не тестировать LLM как обычную deterministic функцию;
- отдельно тестировать retrieval;
- отдельно тестировать prompt behavior;
- добавить негативные вопросы;
- отчёт должен показывать, где pipeline ошибается.

Формат ответа:

## Scope
Файлы.

## Evaluation target
Что проверяем.

## Plan
План.

## Changes
Изменения.

## Metrics
Какие метрики добавлены.

## Test cases
Какие кейсы покрыты.

## How to run
Команда запуска.

## Commit message
Пример:

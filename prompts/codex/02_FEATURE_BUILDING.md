Ты реализуешь новую feature в проекте.

Обязательный режим:

## Scope
Работай только с файлами, указанными в задаче.

## Ignore
Не открывай:
- data/
- notebooks/
- tests/
- logs/
- .venv/
- __pycache__/

## Focus
Сконцентрируйся только на одной feature.

Перед кодом:
1. Опиши, что будет добавлено.
2. Объясни, зачем это нужно в проекте.
3. Дай короткий план.
4. Перечисли файлы, которые будут изменены.

Во время реализации:
- не меняй публичные интерфейсы без причины;
- не добавляй лишние зависимости;
- не смешивай feature с рефакторингом;
- не добавляй тестовые заглушки в production-код;
- не оставляй TODO без объяснения.

После реализации дай отчёт:

## What was implemented
Что реализовано.

## Why it matters
Зачем это нужно для проекта.

## Files changed
Какие файлы изменены.

## Behavior
Как теперь работает система.

## Validation
Как проверить.

## Commit message
Напиши один commit message.
Пример:
feat: add telegram webhook message handling
